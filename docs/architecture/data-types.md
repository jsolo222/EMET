# Data Types

Four types. All information entering or generated by the system is assigned exactly one type at intake. Type can be upgraded or downgraded as evidence accumulates. Downgrades are permanent records.

## VERIFIED (3 Mothers — Aleph, Mem, Shin)

Confirmed by multiple independent sources across different domains with no detectable coordination. Cross-domain confirmation required — same-domain confirmation alone is insufficient. The axiomatic foundation. Non-negotiable ground.

- Confidence floor: HIGH
- Decay rate: Slowest
- Upgrade path: CONTESTED → VERIFIED (requires cross-domain independent confirmation, minimum 3 sources, coordination check must pass)

## CONTESTED (7 Doubles — Bet, Gimel, Dalet, Kaf, Pe, Resh, Tav)

Real phenomenon with credible sourcing. Disputed interpretation or incomplete evidence. Could resolve either direction. Dual nature.

- Confidence floor: MEDIUM
- Decay rate: Standard
- Upgrade: SPECULATIVE → CONTESTED (requires credible sourcing, no primary source contradiction)
- Downgrade: CONTESTED → SPECULATIVE (requires primary source contradiction or source discrediting)

## SPECULATIVE (12 Simples)

Unverified. Circulating. No archival confirmation. Present because absence of confirmation is not confirmation of absence. Investigation seeds only — never surfaced as findings.

- Confidence floor: LOW
- Decay rate: Fastest effective (due to low floor)

## QUERY (Yesod-Tiphareth loop)

Internally generated investigation thread. A question the system asks itself.

- States: OPEN → ESCALATED → RESOLVED or CLOSED
- Does not decay — persists until resolved or closed
- A QUERY closed without findings is itself a finding
